Captures trigger on 3 events: (1) Window change — you switch apps or documents, (2) Idle-to-active — you return from being away 5+ minutes, (3) Periodic — every 60s as a fallback during active use. Each capture includes a screenshot plus rich metadata: which app, which document, how long you were there, what you were doing before, and the browser URL if applicable.

What happens next: Captures queue locally in SQLite. When the pipeline server (GB10) is reachable, they drain automatically. The pipeline runs OCR on screenshots, then an LLM extracts knowledge engrams — summaries, tags, and confidence scores. The metadata you see below makes those extractions significantly richer.
